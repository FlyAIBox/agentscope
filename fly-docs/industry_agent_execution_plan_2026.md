# 基于 AgentScope 的行业智能体项目执行方案

日期：2026-07-08
关联文档：[行业智能体构建平台市场调研](./industry_agent_market_research_2026.md)

## 1. 总体方向

不要一开始做“大而全的智能体平台”。后续应按“样板验证 -> 内容获客 -> MVP 平台 -> 客户项目 -> 行业模板沉淀”的顺序推进。

核心定位：

> 基于 AgentScope，帮助行业客户把知识、数据和业务流程做成可嵌入小程序/网站/企业系统的智能体，并提供持续运营和定制化服务。

早期不要和 Dify、Coze、FastGPT 直接拼通用平台能力，而是做更聚焦的行业交付：

- 有明确业务闭环：咨询、诊断、预约、工单、报告、转人工。
- 有可衡量指标：解决率、转化率、人工介入率、处理时长、报告生成时间。
- 有行业资产沉淀：知识库模板、流程模板、工具连接器、评估集、部署方案。

## 2. 优先级判断

### 第一优先级：电子产品维修/售后智能体

适合作为第一个样板。

原因：

- 场景具体，用户容易理解。
- 流程清晰：故障描述 -> 追问 -> 初步诊断 -> 维修建议 -> 报价区间 -> 预约/工单。
- 风险低于医疗、心理、法律。
- 可以很好展示 AgentScope 的多轮追问、工具调用、RAG、工单创建、人工接管。

第一版不要覆盖所有电子产品，建议只选一个小范围：

- 手机维修。
- 笔记本电脑维修。
- 家电售后。
- 智能硬件/电子产品售后。

建议首选“手机/笔记本维修”，因为故障类型高频、案例多、演示容易。

### 第二优先级：数据分析/经营咨询智能体

适合作为技术展示和企业客户入口。

原因：

- 能体现 AgentScope 的工具执行、沙箱、文件处理、报告生成。
- 适合做直播课和企业内训。
- 客户容易把价值理解成“节省分析师/运营人员时间”。

第一版支持：

- 上传 CSV/Excel。
- 自动识别字段。
- 生成数据概览。
- 找异常、趋势、Top N。
- 输出经营分析报告。
- 支持用户追问。

### 第三优先级：心理健康科普/咨询预约助手

适合作为流量型案例，但不适合作为早期主交付方向。

边界要明确：

- 做心理健康科普。
- 做情绪记录和压力自测。
- 做咨询前信息收集。
- 做咨询师预约和人工转接。
- 不做诊断。
- 不承诺替代心理咨询师。
- 高风险表达必须转人工或提示专业求助。

## 3. 阶段路线

## 阶段 0：准备期，1 周

目标：把方向收窄，确定第一个可演示样板。

输出物：

- 1 个明确样板行业：建议电子产品维修。
- 1 份样板智能体脚本：用户问题、追问路径、知识库、工具调用、最终输出。
- 1 份 AgentScope 技术实现清单。
- 1 份公众号/直播课选题大纲。

要做的事：

1. 选定第一个细分场景，例如“手机维修售后智能体”。
2. 收集 30-50 条典型用户问题。
3. 整理 10-20 个故障类型。
4. 定义智能体必须完成的业务动作。
5. 定义哪些问题必须转人工。

建议不要超过 1 周。这个阶段不是做研究，而是定边界。

## 阶段 1：Demo 样板，2-3 周

目标：做出一个能演示的端到端行业智能体。

Demo 需要覆盖完整链路：

1. 用户输入故障问题。
2. 智能体追问关键信息。
3. 从知识库检索维修手册/FAQ/历史案例。
4. 给出初步诊断和安全操作建议。
5. 判断是否需要人工。
6. 生成维修工单摘要。
7. 模拟预约或报价。
8. 后台能看到会话、工单、知识引用。

技术实现建议：

- AgentScope Agent Service 作为后端服务。
- RAG 接入维修知识库。
- Tool 调用模拟工单系统。
- Permission/人工确认用于高风险动作。
- Event stream 推送智能体执行过程。
- 前端先做简单网页，不急着做完整小程序。

Demo 不需要一开始做复杂 UI。建议先做三个页面：

- 用户对话页。
- 后台会话/工单页。
- 知识库管理页。

验收标准：

- 能连续跑通 10 个典型故障案例。
- 回答能显示知识来源。
- 无法判断时能转人工。
- 能生成结构化工单。
- 能录制成 3-5 分钟演示视频。

## 阶段 2：内容获客，和 Demo 同步推进

目标：用内容验证市场反馈，不等平台完全做好再宣传。

公众号文章建议：

1. 《2026 年行业智能体不是聊天机器人，而是业务流程执行层》
2. 《为什么电子产品维修适合第一个做 AI Agent》
3. 《用 AgentScope 做一个售后维修智能体：从故障诊断到工单生成》
4. 《Dify、Coze、FastGPT、AgentScope 分别适合什么场景》
5. 《企业做智能体最容易踩的 5 个坑：知识库、工具、评估、转人工、成本》

直播课建议：

- 主题：从 0 到 1 构建一个行业智能体。
- 时长：90-120 分钟。
- 结构：
  - 20 分钟讲市场和机会。
  - 30 分钟讲 AgentScope 技术架构。
  - 30 分钟演示维修智能体。
  - 20 分钟讲如何接入网站/小程序。
  - 20 分钟讲客户项目如何报价和交付。

内容目标不是泛泛讲 AI，而是筛选潜在客户：

- 小程序/网站开发商。
- 维修/售后服务商。
- 企业数字化负责人。
- 想接 AI 项目的开发者和顾问。

每篇内容都要导向一个动作：

- 预约演示。
- 加交流群。
- 报名直播。
- 提交自己的行业场景。

## 阶段 3：MVP 平台，4-8 周

目标：把 Demo 变成可以服务 2-3 个试点客户的最小平台。

MVP 必须有：

| 模块 | 第一版功能 |
| --- | --- |
| 租户管理 | 每个客户独立知识库、会话、配置 |
| 智能体模板 | 至少支持维修售后、数据分析两个模板 |
| 知识库 | 文档上传、分段、检索、引用来源 |
| 工具调用 | 工单创建、预约模拟、CSV/Excel 分析 |
| 会话后台 | 查看用户问题、回答、引用、转人工原因 |
| 评估集 | 维护标准问题和标准答案，跑准确率/可用率 |
| 渠道嵌入 | 网站 JS Widget 或 iframe；小程序后置 |
| 成本统计 | 模型调用次数、Token、单次会话成本 |

MVP 可以暂时不做：

- 复杂拖拽编排。
- 应用市场。
- 多模型复杂路由。
- 大而全权限系统。
- 完整 CRM。
- 完整小程序装修后台。

原因：早期真正重要的是跑通行业交付闭环，不是复刻 Dify/Coze。

## 阶段 4：试点客户，8-12 周

目标：拿 2-3 个真实客户试点，形成案例。

客户选择标准：

- 有真实咨询/售后流量。
- 有 FAQ、手册、历史工单或产品资料。
- 愿意配合整理知识库。
- 愿意提供效果指标。
- 能接受“试点版”，不会要求一开始就全功能。

首批客户建议：

1. 本地维修门店或连锁维修品牌。
2. 某个电子产品/智能硬件售后团队。
3. 有内部数据分析需求的小企业或咨询团队。

试点交付流程：

1. 需求访谈：确认目标和边界。
2. 数据收集：FAQ、手册、工单、产品信息。
3. 知识库整理：清洗、分类、标注。
4. Agent 流程配置：追问、诊断、转人工、工单。
5. 测试集建设：至少 50 条标准问题。
6. 上线试运行：先内部用，再小流量开放。
7. 周报复盘：解决率、转人工、错误回答、知识缺口。
8. 迭代优化：每周更新知识库和规则。

试点 KPI：

- 常见问题自动解决率。
- 转人工率下降。
- 平均响应时间下降。
- 工单填写完整率。
- 用户留资/预约转化率。
- 错误回答率。
- 单次会话成本。

## 4. 产品架构建议

### 4.1 后端核心

建议用 AgentScope 做核心智能体运行层：

- Agent 定义：不同场景的角色、系统提示词、工具、知识库。
- Agent Service：对外提供会话 API。
- Multi-session：支持多用户多会话。
- Multi-tenancy：支持多个客户隔离。
- Event System：把执行过程推给前端和后台。
- Middleware：做日志、成本控制、安全审计、评估。
- Workspace/Sandbox：支持数据分析、文件处理、代码执行。

### 4.2 平台层

需要自己补齐：

- 客户后台。
- 模板管理。
- 知识库管理。
- 渠道嵌入配置。
- 会话和工单看板。
- 评估和报表。
- 计费和成本统计。

### 4.3 前端入口

优先级：

1. 网站 Widget：最快上线，适合演示和客户试点。
2. H5 页面：适合公众号和投放。
3. 小程序：适合行业客户正式运营。
4. 企业微信/飞书机器人：适合企业内部知识库和数据分析。

不要第一天就做完整小程序平台。先用网站 Widget 和 H5 验证。

## 5. 样板一：电子产品维修智能体

### 5.1 用户流程

用户问题：

> 我的 iPhone 进水后开不了机，怎么办？

智能体流程：

1. 判断风险：进水、不开机，提示不要充电、不要加热。
2. 追问：型号、进水时间、是否充电、是否有发热、屏幕状态。
3. 检索知识库：进水处理 SOP、常见故障、维修案例。
4. 给出初步判断：可能涉及主板、电池、屏幕排线。
5. 给出建议：停止自行处理，预约检测。
6. 生成工单：型号、故障、风险等级、建议检测项。
7. 引导预约：选择门店/上门/邮寄。

### 5.2 知识库结构

- 产品型号库。
- 常见故障库。
- 维修 SOP。
- 报价区间。
- 保修政策。
- 安全风险提示。
- 历史案例。
- 转人工规则。

### 5.3 工具

- 创建工单。
- 查询报价。
- 查询门店。
- 查询预约时间。
- 发送短信/微信提醒。
- 查询历史维修记录。

第一版工具可以先模拟，后续再接真实系统。

## 6. 样板二：数据分析智能体

### 6.1 用户流程

用户上传销售数据 CSV，提出：

> 帮我分析一下最近三个月销售下滑的原因。

智能体流程：

1. 识别字段：日期、产品、渠道、地区、销售额、订单数、客单价。
2. 数据质量检查：缺失值、异常值、字段类型。
3. 生成概览：总销售额、环比、同比、Top 产品、Top 渠道。
4. 找异常：哪个地区/渠道/产品下降最大。
5. 归因分析：订单数下降还是客单价下降。
6. 生成建议：补货、促销、渠道优化、重点客户回访。
7. 输出报告：Markdown/PDF/网页。

### 6.2 技术重点

- 文件上传。
- 沙箱执行 Python 分析。
- 图表生成。
- 结果解释。
- 多轮追问。
- 报告导出。

### 6.3 价值指标

- 分析报告生成时间。
- 人工分析节省时长。
- 用户追问完成率。
- 报告可采纳建议数。

## 7. 交付报价建议

早期可以采用三档：

| 档位 | 客户 | 内容 | 价格建议 |
| --- | --- | --- | --- |
| 轻量版 | 小商家、个人服务商 | 1 个智能体、1 个知识库、网站嵌入、基础后台 | 3000-10000 元一次性 + 月服务费 |
| 标准版 | 中小企业 | 2-3 个场景、知识库整理、工单/预约工具、月度优化 | 2-8 万元项目费 + 月服务费 |
| 定制版 | 中大型企业 | 私有部署、系统集成、权限、评估、安全审计 | 10 万元起，按需求报价 |

月服务费建议覆盖：

- 模型调用成本。
- 知识库维护。
- 效果评估。
- 问题复盘。
- 小版本迭代。

不要只收一次性开发费。智能体真正有价值的是持续运营。

## 8. 90 天行动计划

### 第 1-7 天

- 确定第一个样板：手机/笔记本维修智能体。
- 整理 50 条典型问题。
- 整理 20 个故障类型。
- 写出完整对话脚本。
- 确定技术架构。
- 写第一篇公众号文章。

### 第 8-21 天

- 搭建 AgentScope 后端 Demo。
- 接入简易维修知识库。
- 实现模拟工单工具。
- 做一个网页对话入口。
- 做一个后台会话页。
- 跑通 10 个典型案例。
- 录制演示视频。

### 第 22-35 天

- 加入引用来源。
- 加入转人工规则。
- 加入基础评估集。
- 写 2-3 篇技术/案例文章。
- 准备第一场直播课。
- 找 5-10 个潜在试点客户访谈。

### 第 36-60 天

- 做 MVP 后台：租户、知识库、模板、会话、工单。
- 增加数据分析智能体 Demo。
- 网站 Widget 可嵌入。
- 拿下 1 个试点客户。
- 按周复盘真实问题。

### 第 61-90 天

- 完成 2-3 个试点。
- 沉淀第一个行业模板包。
- 形成客户案例。
- 明确收费方式。
- 开始小范围销售。
- 根据试点反馈决定是否做小程序端。

90 天结束时应有这些成果：

- 1 个可演示维修智能体。
- 1 个可演示数据分析智能体。
- 1 个基础平台后台。
- 1 个网站嵌入入口。
- 2-3 篇公众号深度文。
- 1 场直播课。
- 1-3 个试点客户。
- 1 套行业模板资产。

## 9. 团队分工

最小团队可以是 2-3 人：

| 角色 | 职责 |
| --- | --- |
| 技术负责人 | AgentScope 后端、RAG、工具调用、平台架构 |
| 产品/交付负责人 | 行业流程、客户访谈、知识库整理、测试集、交付复盘 |
| 内容/增长负责人 | 公众号、视频、直播课、社群、客户线索 |

如果只有一个人做，优先顺序是：

1. Demo。
2. 内容。
3. 试点客户。
4. 后台平台。

不要先闭门做完整平台。

## 10. 决策原则

后续每个功能都用三个问题判断：

1. 这个功能能不能帮助拿到真实客户？
2. 这个功能能不能提升行业交付效率？
3. 这个功能能不能沉淀为可复用资产？

如果答案都是否，就先不做。

早期最容易犯的错误：

- 做成通用 Agent 平台。
- 过早做复杂拖拽编排。
- 过早做小程序完整后台。
- 没有真实客户就做权限、计费、应用市场。
- 只讲技术，不讲业务闭环。
- 只做聊天，不做工单、预约、报告、转人工。
